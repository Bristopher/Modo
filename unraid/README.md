# Unraid deploy — `unraid/`

This folder is what runs on the **NAS**. It uses **prebuilt images** (built on your Windows dev
box), so Unraid never needs the monorepo source. There are two ways to get the images onto the NAS —
pick one:

| Path | Build script | Transfer | Best for |
| --- | --- | --- | --- |
| **A. Private registry** (recommended) | `scripts/build_and_push.ps1` | `docker compose pull` (only changed layers) | Repeat deploys / updates |
| **B. One tar file** | `scripts/build_for_unraid.ps1` | copy `fluxer-images.tar`, `docker load` | First time / no registry |

Both bake your **production domain** into the SPA at build time, so they're domain-specific. Change
the domain → rebuild (see `Docs/12`).

> **The domain is compiled into the frontend.** Editing `config.json` alone does not change it —
> you must rebuild. This is the single most common self-host gotcha.

---

## Files that live here

```
unraid/
  compose.yaml            # the ONLY compose file you run on the NAS (prebuilt images)
  registry-compose.yaml   # optional: stand up your own registry (you already did via CA)
  .env                    # GITIGNORED — image tags + SMTP password (generated by the build script)
  config/                 # GITIGNORED — seeded on the dev box (secrets, voice keys, your domain)
    config.json
    livekit.yaml
    Caddyfile
```

`.env` and `config/` are gitignored because they hold secrets (the Gmail app password, the instance
keyring). The build scripts generate them; you copy them to the NAS out-of-band (not via git).

## Path A — private registry (your setup: `192.168.1.200:5024`)

You already added a Docker Registry on Unraid at **`192.168.1.200:5024`**. One-time setup, then every
deploy is a fast push/pull of only the changed layers.

### One-time: tell Docker Desktop the registry is plain-HTTP

LAN registries serve over HTTP, which Docker refuses by default. On the **Windows dev box**:
Docker Desktop → **Settings → Docker Engine**, add the registry to `insecure-registries`, then
**Apply & Restart**:

```json
{
  "insecure-registries": ["192.168.1.200:5024"]
}
```

(You do **not** need this on the NAS — it pulls from `localhost:5024`, and Docker always treats
`localhost` registries as insecure-OK.)

### Each deploy (from the repo root on Windows)

```powershell
pwsh -File scripts/build_and_push.ps1 -Registry 192.168.1.200:5024
# other domain:  ... -Registry 192.168.1.200:5024 -Domain chat.bigweld.duckdns.org
```

That script: seeds/aligns `config/config.json` for the domain → builds both images → tags & pushes
to `192.168.1.200:5024` → stages `unraid/config/` + `unraid/.env` (with the image refs pointing at
`localhost:5024/...` so the NAS pulls locally).

### On the NAS

Copy `unraid/` to the NAS (e.g. `/mnt/user/appdata/fluxer/`) — only needed the **first** time, or
when `config/` or `compose.yaml` changes. Then:

```bash
docker network create bignet            # first time only, if it doesn't exist
docker compose pull                     # grabs the tags you just pushed
docker compose --profile voice up -d    # drop --profile voice if you don't want voice
```

**Updating later:** re-run `build_and_push.ps1` on Windows, then on the NAS just
`docker compose pull && docker compose --profile voice up -d`. Only changed layers transfer.

## Path B — one tar file (no registry)

```powershell
pwsh -File scripts/build_for_unraid.ps1     # or: VS Code → "Fluxer: Build + Save for Unraid"
```

Produces `unraid/fluxer-images.tar` (multi-GB) plus the staged `config/` + `.env`. Copy the whole
`unraid/` folder to the NAS, then:

```bash
docker load -i fluxer-images.tar
docker network create bignet            # first time only
docker compose --profile voice up -d
```

## Reverse proxy (Nginx Proxy Manager)

NPM must be **on the `bignet` network** so it can reach the Caddy container by name. The Fluxer stack
publishes **no** host port for Caddy — NPM talks to it container-to-container.

Proxy host:
- **Domain Names:** `fluxer.bigweld.duckdns.org`
- **Scheme:** `http`  •  **Forward Hostname:** `caddy`  •  **Forward Port:** `8080`
- **Websockets Support:** ON  •  **Block Common Exploits:** ON
- **SSL:** request a Let's Encrypt cert (DuckDNS DNS challenge), Force SSL ON, HTTP/2 ON
- **Advanced:**
  ```nginx
  client_max_body_size 100M;
  proxy_read_timeout 3600s;
  proxy_send_timeout 3600s;
  ```

### Router port forwards
- **80 / 443 TCP** → your NPM host (for the web app + TLS).
- **Voice only** (if using LiveKit): **7882/udp**, **7881/tcp**, **3478/udp** → the NAS.

## Quick reference

| Want to… | Do this |
| --- | --- |
| Deploy / update (registry) | `build_and_push.ps1 -Registry 192.168.1.200:5024` → on NAS `docker compose pull && up -d` |
| Deploy without a registry | `build_for_unraid.ps1` → copy tar → `docker load` → `up -d` |
| Change the domain | edit `.env` + rebuild (see `Docs/12` §B) — restart alone is **not** enough |
| Back up the instance | copy `config/config.json` + tar the `fluxer_data` volume (`Docs/12` Backups) |
| Run local dev instead | VS Code **Ctrl+Shift+B**, or `scripts/dev_up.ps1` |
